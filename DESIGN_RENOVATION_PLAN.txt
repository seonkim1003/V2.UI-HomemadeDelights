TIMING TOKENS
-------------
  --ease-out:    cubic-bezier(0.25, 0, 0, 1);
  --ease-spring: cubic-bezier(0.34, 1.56, 0.64, 1);
  --ease-bounce: cubic-bezier(0.68, -0.55, 0.265, 1.55);

  --dur-fast:   200ms;
  --dur-normal: 350ms;
  --dur-slow:   600ms;
  --dur-hero:   900ms;

Core rules:
  - ONLY animate transform and opacity - never width/height/top/left
  - Exit animations = 60% of enter duration
  - ALWAYS include @media (prefers-reduced-motion: reduce) override


SCROLL REVEAL SYSTEM (replace current .fade-in system)
-------------------------------------------------------
CSS:
  .reveal-ready {
    opacity: 0;
    transform: translateY(32px);
    transition: opacity var(--dur-slow) var(--ease-out),
                transform var(--dur-slow) var(--ease-out);
  }
  .reveal-ready.revealed {
    opacity: 1;
    transform: translateY(0);
  }

  /* Stagger children automatically */
  .stagger-children .reveal-ready:nth-child(1) { transition-delay: 0ms; }
  .stagger-children .reveal-ready:nth-child(2) { transition-delay: 60ms; }
  .stagger-children .reveal-ready:nth-child(3) { transition-delay: 120ms; }
  .stagger-children .reveal-ready:nth-child(4) { transition-delay: 180ms; }
  .stagger-children .reveal-ready:nth-child(5) { transition-delay: 240ms; }

  @media (prefers-reduced-motion: reduce) {
    .reveal-ready { opacity: 1; transform: none; transition: none; }
  }

JS IntersectionObserver:
  const observer = new IntersectionObserver((entries) => {
    entries.forEach(entry => {
      if (entry.isIntersecting) {
        entry.target.classList.add('revealed');
        observer.unobserve(entry.target);
      }
    });
  }, { threshold: 0.1, rootMargin: '0px 0px -40px 0px' });
  document.querySelectorAll('.reveal-ready').forEach(el => observer.observe(el));

JS logic on internal link click:

  1. Set transform-origin top, scaleY: 1 (no transition, instant)
  2. Then animate scaleY 1->0 (350ms ease-out)

================================================================================
  SECTION 11: WHY THIS DESIGN FEELS HUMAN, NOT AI-GENERATED
================================================================================

These specific choices are what separate this from auto-generated templates:

1. GRAIN TEXTURE OVERLAY
   No AI design tool, Wix template, or theme generator adds film grain.
   It makes screens feel tactile, printed, and physical.
   Instantly elevates perceived quality without any structural changes.

2. ASYMMETRIC BENTO GRID (social section)
   AI generators always output uniform 3-column equal-size grids.
   Intentional size variation (one large + two small) signals a human
   designer who consciously chose visual hierarchy over convenience.

3. PARALLELOGRAM HERO IMAGE CROP
   clip-path: polygon(8% 0%, 100% 0%, 92% 100%, 0% 100%)
   No auto-generated site uses clip-path on a hero image.
   It breaks the box. It is impossible to mistake for a template.

4. CALISTOGA FONT CHOICE
   Most AI-suggested display fonts: Playfair Display, DM Serif, Merriweather.
   Calistoga has ink-trap warmth, slight imperfection, and unexpected
   personality. Looks like a designer scrolled through 200 fonts and
   stopped on the right one - because they did.

5. TEXT REVEAL (overflow:hidden + translateY mask)
   Used by Linear, Vercel, Stripe, and top design studios worldwide.
   Cheap sites and AI generators use opacity fade-in only.
   The mask reveal screams craft and intentionality.

6. WARM NEAR-BLACK (#1A1512)
   The darkest tone has warmth in it (red-brown channel present).
   This ties the terracotta palette together at the shadow level.
   AI systems always generate cold blue-blacks like #111827.
   This one small number change affects every shadow on the site.

7. PAGE CURTAIN TRANSITION
   A full-page terracotta wipe between every page navigation.
   Almost zero templates or AI-generated sites do inter-page transitions.
   It creates brand identity through motion - the color means something.

8. ORGANIC BLOB PHOTO CROPS (team cards)
   border-radius: 60% 40% 70% 30% / 50% 60% 40% 50%
   Not circles. Not squares. Not rounded squares.
   Looks like it was cut by hand. Feels alive and personal.

9. DECORATIVE BACKGROUND TEXT ("HOME" in hero)
   A technique from print design and editorial layout.
   Signals editorial thinking rather than web-template thinking.
   AI generates hero content that IS the text - not texture.

10. FILM STRIP CAROUSEL AESTHETIC
    Sprocket holes + dark background + edge-peek effect on about page.
    Completely unexpected in a community nonprofit context.
    When design surprises you in a good way, it becomes memorable.

11. TERRACOTTA + SAGE GREEN palette
    The default AI nonprofit palette: blue + white + orange CTA.
    Terracotta + sage = warmth, craft, growth, community, and rarity.
    It feels earned, not generated.

12. STATS COUNT-UP ANIMATION
    Numbers that count from 0 to their final value when scrolled into view.
    A small detail. Templates never do this. It shows intentionality.
    Every small moment of delight compounds into a feeling of quality.
